## All Animations Are CSS-Only with No External Dependencies

### Preconditions
- Dashboard is running with an active pipeline build
- All animations are visible (spinner, progress bar, run card pulse)

### Test Steps
1. Open dashboard source (view-source or browser DevTools)
2. Search the full HTML source for: <link>, external stylesheet references, .gif, .svg animation, requestAnimationFrame, anime.js, gsap, framer-motion, or any JS animation library
3. Inspect the <style> block — verify all @keyframes are defined inline
4. Verify no <script> tags reference animation libraries
5. Check that spinner elements use CSS animation property, not JS-driven style changes

### Expected Results
- ALL animations are defined within the inline <style> block (generated by generateStyles())
- No external CSS files loaded for animations
- No external image assets (GIFs, animated SVGs) used
- No JavaScript animation libraries imported or referenced
- No requestAnimationFrame() calls for animation purposes
- All animated elements use CSS 'animation' property referencing @keyframes defined in the <style> block
- The only existing @keyframes ('pulse' for auto-refresh) is preserved and not modified
- New @keyframes have distinct names (not 'pulse')
- Total added CSS is reasonable (under 100 lines of animation CSS)

### Pass/Fail Criteria
- PASS: All animations are pure CSS within inline <style> block, no external dependencies
- FAIL: Any external asset, JS animation library, or animation logic outside of CSS
