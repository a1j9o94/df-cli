How it works

An 8-phase pipeline that takes a specification and produces validated, merged code.

scout →
architect →
plan-review →
build →
integrate →
evaluate →
merge
01

Scout

Index the codebase so agents understand the project structure, conventions, and patterns.

02

Architect

Decompose the spec into modules with interface contracts, a dependency DAG, and integration strategy.

03

Plan Review

Auto-approve small plans. Present complex decompositions for human review before building.

04

Build

Spawn parallel builders, each in its own git worktree, bound by typed interface contracts.

05

Integrate

Merge worktrees and run checkpoint tests to verify modules compose correctly.

06

Evaluate

Run holdout scenarios the builders never saw. Test changeability with fresh builders.

07

Iterate

If evaluation fails, retry from build phase with feedback. Up to max_iterations attempts.

08

Merge

Integrate into the target branch with post-merge validation. Ship it.

Key features

The primitives that make parallel AI builds reliable.

⚓

Parallel Builders

Multiple AI builders work simultaneously in isolated git worktrees. No shared mutable state. Integration happens after builds complete.

🔍

Holdout Scenarios

Test scenarios builders never see. Validate what the human actually wants, not what the builder thought they wanted.

📜

Contract-Driven Decomposition

Typed interface contracts define module boundaries. Builders implement to contracts without coordinating with each other.

💰

Budget Control

Every agent reports cost. The pipeline respects budgets, surfaces tradeoffs, and prevents runaway spending.

🔀

TDD Cycles

Builders follow red-green-refactor cycles. Tests are written first, then minimal code to pass, then cleanup.

📨

Agent Mail System

SQLite-backed messaging between agents. Durable, inspectable, debuggable. No direct process communication.

CLI examples

Everything you need from the command line.

Build a spec 
# Create a spec
$ dark spec create "Add JWT authentication"

# Run the full pipeline
$ dark build spec_01ABC123

# Watch progress
$ dark status
Dashboard & monitoring 
# Launch the live dashboard
$ dark dash

# Check agent status
$ dark agent list --run-id run_01XYZ

# View pipeline phases
$ dark status --json
Architecture & contracts 
# Analyze a spec
$ dark architect analyze spec_01ABC123

# View interface contracts
$ dark contract list --run-id run_01XYZ

# Check contract compliance
$ dark contract check ctr_01DEF

Get started

Up and running in under a minute.

1.

Clone the repo

git clone https://github.com/a1j9o94/df-cli.git && cd df-cli
2.

Install dependencies

bun install
3.

Initialize your project

dark init --name my-project
4.

Create a spec and build

dark spec create "My feature" && dark build <spec-id>
